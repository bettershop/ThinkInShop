## 实现原理 => 专属打包文件夹 → 编译到 node_modules → 小程序不编译原文件夹


## 实现步骤
## packages 文件夹存放公共文件（工具类 / 组件 / 配置）
## 编译 packages 到 node_modules/@my-miniprogram（自定义命名空间，避免冲突）
## 微信开发者工具编译时跳过 packages 文件夹，仅使用 node_modules 中的编译后文件


## 文件结构说明
## packages : 专属打包文件夹（存放公共文件）
## scripts : 构建脚本目录
## scripts / build-packages.js : # 编译packages到node_modules的脚本
## 
## 执行方式
## 打包运行指令 （将 packages文件 编译到 node_modules包中）
## 
##  npm run build:packages
## 
##  当看到 黑窗口/控制台  提示 "公共文件已成功编译"  则代表着 文件已经成功编译到 node_modules包中
