# 生产上线检查清单（集群）

## 1. 发布前（T-1 天 / T-2 小时）

### 1.1 资源与连通性

1. RDS 白名单已放开应用节点网段。
2. Redis Cluster 白名单已放开应用节点网段。
3. OSS 子账号权限仅限目标 Bucket。
4. SLS 采集器在线，能看到测试日志。

### 1.2 配置与密钥

1. 已完成 `docs/15、集群生产参数清单（待填）.md` 参数填写。
2. K8s Secret 或 ECS 环境变量已注入数据库/Redis/OSS 密钥。
3. 应用配置已指向生产 RDS 与 Redis Cluster。
4. WebSocket 入口配置（Nginx/Ingress）包含 Upgrade 头与超时设置。

### 1.3 包与镜像

1. 代码分支已冻结，发布 commit 已确认。
2. 镜像标签为不可变版本（如 `v20260323-01`）。
3. 回滚镜像标签已确认可拉取。
4. 预发布环境已通过核心流程验证。

### 1.4 功能验收（预发）

1. 用户登录、下单、支付回调链路正常。
2. 文件上传后能在 OSS 查询到对象。
3. IM WebSocket 在多副本下可互通（跨实例发送成功）。
4. 核心后台功能无 5xx 错误。

## 2. 发布中（T 时刻）

### 2.1 执行步骤

1. 部署新版本（滚动发布，不全停）。
2. 观察每个新实例健康检查通过再继续下一批。
3. 发布期间冻结高风险数据库结构变更。

### 2.2 实时观察项

1. `HTTP 5xx` 错误率。
2. 平均响应时间与 P95/P99。
3. JVM 堆使用率、Full GC 次数。
4. RDS 连接数与慢 SQL。
5. Redis 延迟与拒绝连接数。
6. SLS 错误日志突增。

## 3. 发布后（T+30 分钟）

### 3.1 核心功能回归

1. 新用户注册与登录。
2. 商品浏览与下单。
3. 支付回调与订单状态流转。
4. OSS 上传与访问链接。
5. IM 消息收发与在线状态更新。

### 3.2 集群专项

1. 随机抽 2 个实例，确认均有流量。
2. 缩容/扩容 1 次，确认服务无中断。
3. WebSocket 长连接 30 分钟保持稳定。
4. SLB/Ingress 无异常断连告警。

## 4. 回滚预案（必须预先演练）

### 4.1 回滚触发条件

1. 错误率连续 5 分钟超阈值。
2. 核心交易链路不可用。
3. 关键外部依赖（RDS/Redis）出现严重故障且新版本放大风险。

### 4.2 回滚动作

1. 回滚 Deployment 镜像到上一稳定版本。
2. 验证 `/apis/actuator/health` 与关键接口。
3. 观察 10 分钟确认恢复。
4. 冻结新版本，进入故障复盘。

## 5. 发布完成判定

满足以下条件可判定发布完成：

1. 关键业务流程全绿。
2. 错误率与响应时间恢复到基线区间。
3. SLS 无持续新增高危错误。
4. 运维、开发、业务三方确认。
